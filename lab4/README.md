# Lab 4

## Опис проєкту
Це мобільний менеджер файлів на React Native (Expo). Проєкт демонструє роботу з асинхронним API файлової системи, реалізацію CRUD-операцій та аналіз стану пам'яті девайса.

**Основний функціонал:**
* **Аналітика сховища:** Моніторинг вільного та загального обсягу пам'яті.
* **Файловий менеджер:** Перегляд вмісту папок, перехід по дереву директорій.
* **Робота з даними:** Створення папок, а також написання та редагування .txt файлів.
* **Метадані:** Відображення розміру, розширення та останньої дати зміни об'єктів.

## Інструкція із запуску
1. Клонувати репозиторій:
`git clone https://github.com/Dmytriy-OL/MobileLabsRN2026`

2. Перейти в папку лабораторної роботи:
`cd lab4/my-app`

3. Встановити залежності:
`npm install`

4. Запустити проєкт:
`npx expo start -c`

## Методи тестування проєкту

### 1.Тестування через Expo Go
Основний метод перевірки на реальних пристроях. Дозволяє отримати повноцінний доступ до нативної файлової системи (documentDirectory), що неможливо зробити у звичайному веб-браузері. Для запуску необхідно відсканувати QR-код через мобільний додаток Expo Go.

### 2. Відладка та моніторинг
Контроль за виконанням CRUD-операцій здійснюється через термінал розробника. Усі системні логи, звіти про успішне створення файлів або помилки доступу до пам'яті виводяться в реальному часі для швидкої діагностики коду.

## Технічна реалізація та використані API
* **Модуль ExpoFileSystem (Legacy):** Робота з файлами через методи readDirectoryAsync (читання папок), writeAsStringAsync (запис даних) та deleteAsync (видалення).
* **Метрики сховища:** Використання системних викликів getTotalDiskCapacityAsync та getFreeDiskStorageAsync для аналізу пам'яті.
* **Оптимізація рендеру:** Застосування хука useCallback для мемоізації функції оновлення даних, що запобігає зайвим перемальовуванням інтерфейсу.
* **State Management:** Використання об’єктного стану (viewState) для керування видимістю модальних вікон створення, редагування та перегляду інфо.
* **Path Processing:** Реалізація логіки очищення та форматування шляхів для коректної навігації (наприклад, перевірка закінчення рядка на /).

Під час виконання роботи я опанував методи взаємодії з нативною файловою системою через expo-file-system, реалізувавши повноцінний цикл маніпуляції даними (CRUD) та систему моніторингу дискового простору. Застосування хуків useCallback та useEffect дозволило оптимізувати оновлення інтерфейсу при навігації по каталогах, а використання модальної архітектури забезпечило зручну взаємодію з метаданими та вмістом файлів. У результаті створено стабільний застосунок, що коректно обробляє асинхронні запити до сховища та візуалізує системну інформацію про пам'ять пристрою.
